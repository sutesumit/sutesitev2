## The Itch

> "Some thoughts don't need 2000 words. They just need to exist."

I wanted something like X (Twitter). But smaller. Mine. A place to dump quick thoughts without the friction of opening a browser, logging in, composing, clicking post. The thoughts I wanted to capture were the fleeting ones — the ones that arrive mid-debug and vanish if not caught.

The requirement was simple: from any terminal, any computer, type a short command and have it appear on my site. No apps, no browser tabs, no context switching.

I called it **blip**.

---

## The Plan

The agent and I sketched it out:

```
Terminal → curl → API → Supabase → /blip page
```

Minimal. The entire flow:

1. **Storage**: A Supabase table called `blips`
2. **API**: A single endpoint at `/api/blip` with POST and GET
3. **Auth**: A secret key in a header (nothing fancy, just enough to keep strangers out)
4. **Frontend**: A `/blip` page that lists them all

The agent proposed options for the CLI command. I wanted it short enough to remember. We settled on:

```bash
curl -X POST https://www.sumitsute.com/api/blip -H "K: secret" -d "thought here"
```

## The Build

The structure came together quickly:

```
src/app/
├── api/blip/route.ts      # POST and GET handler
├── blip/
│   ├── page.tsx           # The feed
│   └── components/
│       ├── BlipCard.tsx   # Individual blip
│       ├── IntroCard.tsx  # Hover card wrapper
│       └── IntroText.tsx  # Quirky intro
├── lib/blip.ts            # Server-side fetch
└── types/blip.ts          # TypeScript types
```

The API was straightforward. A POST handler that validates a key, parses content, inserts into Supabase. A GET handler that returns them all, newest first.

## The First Redirect

I deployed. I tried to post:

```bash
curl -X POST https://sumitsute.com/api/blip -H "X-Key: blip" -d "content=First blip!"
```

Response:

```
Redirecting...
```

Not an error. Not a 404. Just... redirecting. I tried again. Same thing. I added `-v` for verbose output:

```bash
curl -v -X POST https://sumitsute.com/api/blip ...
< HTTP/2 308
< location: https://www.sumitsute.com/api/blip
```

Ah. My site redirects `sumitsute.com` to `www.sumitsute.com`. Curl doesn't follow redirects for POST requests by default.

The fix was simple:

```bash
curl -X POST https://www.sumitsute.com/api/blip ...
```

---

## The JSON Error

First attempt with the correct URL:

```
{"error":"Unexpected token 'c', \"content=First blip!\" is not valid JSON"}
```

The API expected JSON. Curl sent form-urlencoded. I could either:

1. Force users to send JSON (`-H "Content-Type: application/json" -d '{"content":"..."}'`)
2. Make the API accept both

Option 2 was clearly better. I updated the `parseContent` function to handle `application/x-www-form-urlencoded`, JSON, and raw text.

---

## The Ghost Blip

After fixing the API, I posted successfully:

```json
{"blip":{"id":"abc123","content":"Second blip","created_at":"2026-03-05T18:26:12+00:00"}}
```

`201 Created`. Perfect. I refreshed `/blip` to see it.

Nothing.

The page was empty. No blips. But the API returned success. I checked Supabase directly — the blip was there.

The problem: Next.js cached the page at build time. It was static. New blips wouldn't appear until the next deployment.

One line fix:

```typescript
export const dynamic = 'force-dynamic';
```

Now the page fetches fresh data on every visit.

## The Long Command

The working command was:

```bash
curl -X POST https://www.sumitsute.com/api/blip -H "X-Key: blip" -d "content=thought"
```

Functional but verbose. I wanted shorter. The agent suggested an alias, which I'll set up on my machines. But for raw curl from any computer, I wanted the command itself to be minimal.

Two changes:

1. **Header**: `X-Key` → `K` (single letter)
2. **Body**: Accept raw text, no `content=` prefix

The final command:

```bash
curl -X POST https://www.sumitsute.com/api/blip -H "K: blip" -d "thought"
```

Eleven characters of overhead. The rest is the thought itself.

---

## The Alias

For my own machines, I added this to `.bashrc`:

```bash
blip() {
  curl -X POST https://www.sumitsute.com/api/blip -H "K: blip" -d "$*" --silent && echo "✓"
}
```

Now posting a thought is just:

```bash
blip just had an idea about the caching layer
```

---

## What I Took Away

- **The redirect trap.** When testing APIs with curl, always check verbose output. The server might be redirecting you somewhere you didn't expect.
- **Static by default.** Next.js pages are static unless you tell them otherwise. For dynamic content, `force-dynamic` is your friend.
- **Shorten the path.** The goal was minimal friction. Every character in that curl command is friction. The alias removes it entirely on my machines; the shortened raw command keeps it manageable elsewhere.
- **API flexibility.** Accepting multiple content types (form-urlencoded, JSON, raw) makes the API usable from more places with less ceremony.
